# TASK ARCHIVE: AGENTS.md Plugin (`@a16njs/plugin-agentsmd`)

## SUMMARY

Added the bundled included plugin `@a16njs/plugin-agentsmd` ([Issue #50](https://github.com/Texarkanine/a16n/issues/50)): discover `AGENTS.md` files at any directory depth (root → GlobalPrompt, nested → dir-scoped FileRule), emit GlobalPrompts and dir-shaped FileRules back to directory-structured `AGENTS.md` files (lossy; conveyed exclusively through a16n's standard warning channels). Registered in the CLI alongside the other included plugins; release-please, codecov, CI, and full docs wired. All gates green: 36 unit + 4 integration tests for the new surface, 951 tests monorepo-wide, docs build, QA PASS with only trivial findings.

## REQUIREMENTS

### User Story

As a developer working in repositories that use the AGENTS.md standard (or considering it), I want a16n to translate between AGENTS.md files and other agent customization formats, so that I can programmatically enter or — more importantly — exit the AGENTS.md standard.

### Discovery (primary case)

- Discover `AGENTS.md` files at **any** directory depth in a repository.
- Root `AGENTS.md` → GlobalPrompt; nested `AGENTS.md` → FileRule with dir-shaped globs (enables the "escape hatch" into Cursor `globs:` rules and Claude `paths:` rules).
- Skip dot-directories and `node_modules` (mirrors plugin-claude).

### Emission (lossy case)

- GlobalPrompts → root `AGENTS.md` (always-apply semantics); nested-CLAUDE-style GlobalPrompts (`metadata.nested` + `sourcePath`) → `dirname(sourcePath)/AGENTS.md`.
- Dir-shaped FileRules (`<dir>/**` or `<dir>/**/*`) → `<dir>/AGENTS.md`; all other FileRules → `Skipped`.
- Multiple items targeting the same file concatenated with `\n\n`; `Merged` warning when >1 item.
- Deterministic overwrite of existing targets; `Overwritten` warning when content changes (suppressed when byte-identical).
- Non-GlobalPrompt/FileRule types → `unsupported` array.
- **No editorial warnings** about AGENTS.md; no `--force` gating.

### Integration

- Included plugin (ships with a16n), registered in CLI, documented consistently with plugin-claude and plugin-cursor.

## IMPLEMENTATION

### Architecture Decision: IR Mapping for Nested AGENTS.md

**Requirements ranked:** (1) semantic fidelity — nested AGENTS.md means subtree-scoped instructions; (2) escape hatch with zero changes to existing plugins; (3) IR durability through `.a16n/` round-trips; (4) scope discipline — no cursor/claude changes.

**Options evaluated:**

| Criterion | A: FileRule globs | B: GlobalPrompt+relativeDir | C: New IR concept |
|-----------|-------------------|------------------------------|--------------------|
| Fitness (scoped output in cursor/claude) | ✅ `globs:`/`paths:` today | ❌ emits always-apply rules | ✅ but requires all-plugin support |
| Plugin changes required | None | None (but output wrong) | models + all plugins |
| IR durability | ✅ globs+relativeDir serialized | ✅ relativeDir serialized, ❌ semantics lost | ✅ after IR version bump |
| Risk / blast radius | Low, plugin-local | Medium (silent semantic corruption) | High |

**Decision:** Option A — root `AGENTS.md` → `GlobalPrompt(name: 'AGENTS')`; nested `AGENTS.md` → `FileRule(globs: ['<dir>/**'], relativeDir: '<dir>')`.

**Rationale:** Only option satisfying semantic fidelity and escape hatch via existing emitters with zero changes outside the new plugin. `FileRule.globs` is the only existing IR channel producing path-scoped output in both Cursor and Claude. Setting `relativeDir` on discovered FileRules nests emitted rules as `.cursor/rules/<dir>/AGENTS.mdc` (mirrors repo structure).

**Emission placement matrix:**

| IR item | Placement | Warning |
|---|---|---|
| GlobalPrompt (plain; `relativeDir` deliberately ignored) | root `AGENTS.md` (concatenated) | `Merged` if >1 item per file |
| GlobalPrompt with `metadata.nested === true` + `sourcePath` | `dirname(sourcePath)/AGENTS.md` | — |
| FileRule, globs = single dir-shaped pattern | `<dir>/AGENTS.md` | — |
| FileRule otherwise | not written | `Skipped` |
| SimpleAgentSkill / AgentSkillIO / ManualPrompt / AgentIgnore | not written | `unsupported` array |
| Existing target file whose content changes | overwritten | `Overwritten` (first use of this code) |

**Tradeoff:** Provenance is approximate — a FileRule with glob `src/**` from Cursor and a nested `src/AGENTS.md` become indistinguishable in IR (by design: same meaning). Nested-CLAUDE placement relies on transient metadata and degrades to root concatenation after IR round-trips.

### Architecture Decision: Emission Idempotency & Overwrite

**Context:** Issue #50 flags "text concatenation is an idempotency nightmare." AGENTS.md is user-authored at canonical locations (unlike tool-managed `.cursor/rules/` trees).

**Options:** (A) deterministic overwrite + `Overwritten` warning; (B) merge into existing content; (C) refuse to write when target exists.

**Decision:** Option A — emission is a pure function of the IR; `Overwritten` warning iff pre-existing content actually changes (suppressed when byte-identical).

**Rationale:** Only A is simultaneously idempotent, precedent-consistent (every emitter overwrites its own outputs), and honest about data loss. Markdown has no reliable chunk identity, killing merge/dedupe without provenance markers. `WarningCode.Overwritten` existed unused in `@a16njs/models` since inception — this is its first legitimate use.

**Write semantics:** `content.trim()` per item, joined `\n\n`, trailing newline; input order; no separators/markers.

## LESSONS LEARNED

### Technical

- Workspace-internal plugins are resolved via `dist/` by dependent packages' tests; rebuild the plugin after changing `src/` before manually running a dependent's vitest.
- `WarningCode.Overwritten` designed ahead of need paid off — no models change required.
- `generate-versioned-api.ts` try/catches each path checkout, so adding a new package to `WORKSPACE_PACKAGE_PATHS` is safe for historical tags where the package doesn't exist.
- IR-durability analysis was the linchpin: plugin-a16n strips `sourcePath`/`metadata` but keeps `globs`/`relativeDir` — round-trip test designed around globs, not metadata.

### Process

- Mid-task conversation summaries can drift from file reality (regex misquote during QA). QA's value was re-reading shipped code with fresh eyes.
- Preflight listing-sweeps for new packages should grep `packages/docs/docs/**` in addition to root-level READMEs (fifth touchpoint hid in docs intro).
- Both creative decisions (FileRule IR mapping; deterministic-overwrite emission) held with zero friction — no rework required.
